# Story 8.2: 移动端告警历史UI优化

## Status
Done

## Story

**As a** WebSocket中继平台的移动端用户,
**I want** 在移动设备上获得优化的告警历史浏览体验,包括卡片式布局、左滑操作、下拉刷新等移动端原生交互,
**so that** 我可以在手机上更便捷地查看和处理告警,无需依赖PC端登录

## Acceptance Criteria

1. 移动端告警历史页面使用卡片布局，显示流畅美观
2. 左滑操作正常，可以快速标记已读或删除
3. 下拉刷新功能正常，加载动画流畅
4. 底部操作栏固定显示，筛选面板从底部弹出
5. 告警级别颜色条清晰可辨，相对时间显示准确
6. 空状态显示友好提示
7. PC 端显示不受影响，仍使用表格布局

### 移动端卡片布局设计

**MobileAlertCard 组件设计**:
[Source: Epic 8 - Story 8.2 核心功能]

```tsx
// packages/frontend/src/components/endpoints/MobileAlertCard.tsx

import { Card, Tag, Space } from 'antd';
import { formatDistanceToNow } from 'date-fns';
import { zhCN } from 'date-fns/locale';
import { useSwipeable } from 'react-swipeable';
import type { AlertHistoryWithDetails } from '@websocket-relay/shared';

interface MobileAlertCardProps {
  alert: AlertHistoryWithDetails;
  onMarkAsRead: (id: string) => void;
  onDelete: (id: string) => void;
}

function MobileAlertCard({ alert, onMarkAsRead, onDelete }: MobileAlertCardProps) {
  const handlers = useSwipeable({
    onSwipedLeft: () => setShowActions(true),
    onSwipedRight: () => setShowActions(false),
  });

  // 告警级别颜色映射
  const levelColorMap = {
    info: '#1890ff',
    warning: '#faad14',
    critical: '#ff4d4f',
  };

  // 相对时间格式化
  const relativeTime = formatDistanceToNow(new Date(alert.triggered_at), {
    addSuffix: true,
    locale: zhCN,
  });

  return (
    <div {...handlers} style={{ position: 'relative' }}>
      {/* 左侧告警级别色条（4px 宽） */}
      <div
        style={{
          position: 'absolute',
          left: 0,
          top: 0,
          bottom: 0,
          width: 4,
          backgroundColor: levelColorMap[alert.alert_level],
        }}
      />

      <Card
        size="small"
        style={{
          marginBottom: 12,
          paddingLeft: 16, // 为色条留空间
        }}
      >
        {/* 顶部：规则名称 + 状态标签 */}
        <Space style={{ width: '100%', justifyContent: 'space-between', marginBottom: 8 }}>
          <strong>{alert.alert_rule?.rule_name || '-'}</strong>
          <Tag color={alert.status === 'unread' ? 'red' : 'blue'}>
            {alert.status === 'unread' ? '未读' : '已读'}
          </Tag>
        </Space>

        {/* 中间：设备 + 数据字段 + 触发值/阈值 */}
        <div style={{ fontSize: 14, color: '#666', lineHeight: '20px' }}>
          <div>设备: {alert.device?.custom_name || alert.device?.device_id || '-'}</div>
          <div>
            {alert.data_key}: {alert.triggered_value} (阈值: {alert.threshold})
          </div>
        </div>

        {/* 底部：相对时间 */}
        <div style={{ fontSize: 12, color: '#999', marginTop: 8 }}>
          {relativeTime}
        </div>
      </Card>

      {/* 左滑显示的操作按钮（绿色已读 + 红色删除） */}
      {showActions && (
        <div style={{ position: 'absolute', right: 0, top: 0, display: 'flex' }}>
          <button onClick={() => onMarkAsRead(alert.id)}>已读</button>
          <button onClick={() => onDelete(alert.id)}>删除</button>
        </div>
      )}
    </div>
  );
}
```

### 时间格式化工具函数

**时间显示优化**:
[Source: Epic 8 - Story 8.2 核心功能]

```typescript
// packages/frontend/src/utils/time.util.ts

import { formatDistanceToNow, format } from 'date-fns';
import { zhCN } from 'date-fns/locale';

/**
 * 移动端时间格式化
 * 7 天内：相对时间（如"5 分钟前"）
 * 7 天外：绝对时间（如"2025-10-28 14:30"）
 */
export function formatTimeForMobile(date: Date | string): string {
  const dateObj = typeof date === 'string' ? new Date(date) : date;
  const now = new Date();
  const diffInDays = (now.getTime() - dateObj.getTime()) / (1000 * 60 * 60 * 24);

  if (diffInDays < 7) {
    // 7 天内显示相对时间
    return formatDistanceToNow(dateObj, { addSuffix: true, locale: zhCN });
  } else {
    // 7 天外显示绝对时间
    return format(dateObj, 'yyyy-MM-dd HH:mm', { locale: zhCN });
  }
}

/**
 * PC 端时间格式化（保持原有格式）
 */
export function formatTimeForDesktop(date: Date | string): string {
  const dateObj = typeof date === 'string' ? new Date(date) : date;
  return format(dateObj, 'yyyy-MM-dd HH:mm:ss', { locale: zhCN });
}
```

### 编码标准

**关键规则**:
[Source: docs/architecture/coding-standards.md]

1. **Type Sharing**: 所有共享类型定义在 `packages/shared/src/types`
2. **API Calls**: 前端永远通过 `services/` 层调用 API，禁止直接使用 Axios
3. **State Updates**: 前端禁止直接修改状态，使用 setState 或 Context API
4. **Environment Variables**: 通过 `config/` 模块访问环境变量，禁止直接使用 `process.env`

**命名规范**:
- 组件: PascalCase（如 `MobileAlertCard.tsx`）
- Hooks: camelCase with 'use'（如 `useBreakpoint`）
- 函数: camelCase（如 `formatTimeForMobile()`）
- 常量: UPPER_SNAKE_CASE（如 `MIN_TOUCH_TARGET_SIZE = 44`）

**响应式布局性能优化**:
[Source: React 性能最佳实践]
